# Phase 09 - International SEO Expansion

## Context Links
- existing i18n foundation

## Overview
- Priority: P1
- Status: proposed
- Brief: Scale locale only after EN proves templates, governance, and conversion.

## Key Insights
- Locale growth is multiplicative risk:
  - content
  - hreflang
  - canonical
  - QA
  - freshness
- Most sites over-expand locales before proving one strong market.

## Expansion Order
1. EN harden first
2. VI second
3. PT-BR third
4. others only after proof and editorial capacity

## Locale Gating Rules
- proven EN page template
- measurable demand in target locale
- human-edited content available
- locale-specific metadata ready
- hreflang tested
- internal links localized

## Requirements
- Functional:
  - localized content registry
  - canonical master vs locale variant mapping
  - locale sitemap
  - locale performance dashboard
- Non-functional:
  - no raw machine translation publishing
  - no locale with zero ongoing maintenance capacity

## Implementation Steps
1. Build locale-ready content model.
2. Select top 5-10 EN pages to localize first.
3. Create locale QA checklist.
4. Rollout VI on proven pages only.
5. Rollout PT-BR if VI process works.

## Success Criteria
- Locale expansion increases organic reach without creating thin-content footprint.

## Risk Assessment

- stale translations
